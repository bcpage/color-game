NULL — Project & Room Index
===========================
Last updated: 2026-05-22

================================================================================
PROJECT OVERVIEW
================================================================================

NULL is a self-hosted multiplayer web platform built as a labyrinth of 52
self-contained "rooms," each a different experience. The aesthetic anchor is
SAP enterprise software — bureaucratic, institutional, clinical. Players are
tracked before they know they're being tracked. The system does not explain
itself. Navigation is directional (up/down/left/right) via a matrix data
structure. There is no map. Lostness is the design.

Repo: github.com/bcpage/NULL
Stack: Node.js (no framework), vanilla HTML/CSS/JS per room, WebSockets (ws),
       JSON files for all persistence.
Server: server.js — single file, raw HTTP + WebSocket
Port:   3000

================================================================================
ADMIN / SYSTEM PAGES (hidden by URL obscurity)
================================================================================

  /admin        User Registry
                  ALV-grid style table of all tracked subjects. Columns:
                  Cookie ID, IP, First Seen, First Seen (Unix), Last Seen,
                  Visits, Clicks, Browser, OS, Device, Language, Name,
                  Rooms Visited. Per-row delete with confirm. Clear All.
                  Auto-refreshes every 30 seconds. Current user row
                  highlighted green. URL known only to admin.

  /matrix       Matrix Editor
                  Visual editor for the room connection graph. 52 tiles
                  (00000-00051) in a grid. Click FROM room, click TO room,
                  pick direction (up/down/left/right), connect. Inspector
                  panel shows exits, entrances, one-way edges. Dead-end
                  tiles dimmed. Live occupancy from /api/users. Auto-
                  refreshes every 15 seconds. Clear All Connections button.

================================================================================
NAVIGATION SYSTEM
================================================================================

All rooms use data-nav="matrix" on the body tag (except 00000 and 00002).
nav.js dispatches to matrix-nav.js which renders a 3x3 button grid:

      [ ↑ ]
  [←] [ ] [→]
      [ ↓ ]

Buttons disabled if no connection in that direction. Keyboard: WASD + arrows.
Blocked direction = red flash overlay, no navigation.
Connections stored in data/matrix.json (112 connections as of this writing).
Cookie gate (00002) blocks uncookied users from accessing any game room.

================================================================================
USER TRACKING
================================================================================

Every HTTP request is fingerprinted. Data stored in data/users.json keyed
by cookie UUID (set as "device" cookie on first visit, 1-year expiry).

Fields per user:
  id           — UUID (v4), the browser cookie value
  firstSeen    — Unix timestamp of first request
  lastSeen     — Unix timestamp of most recent request
  visits       — total page loads
  cookieClicks — number of times clicked in room 00002
  name         — chosen designation (set in room 00051), null until set
  aliases      — array of previous designations (auto-saved on name change)
  ip           — IP address of most recent request
  browser      — parsed from User-Agent (Chrome, Firefox, Safari, etc.)
  os           — parsed from User-Agent (Windows, macOS, iOS, Android, etc.)
  device       — parsed from User-Agent (Desktop, Mobile, Tablet)
  language     — from Accept-Language header
  rooms        — array of room IDs visited, in order
